User Profile Page

Technical Description

The User Profile Page is structured using semantic HTML elements such as section, div, h3, ul, and form to enhance accessibility and maintainability. The layout follows a flexbox-based CSS design, ensuring a responsive and visually consistent appearance. The profile section uses an img tag for the avatar, styled with border-radius for a circular effect. Skill bars utilize nested div elements with width set dynamically via inline styles. The availability calendar is created using a grid-based layout, aligning days and highlighting availability. The contact form ensures accessibility with required fields and ARIA attributes. A global CSS file maintains design consistency.

Accessibility

In the User Profile Page, I addressed accessibility through several techniques to ensure that all users, including those with disabilities, can navigate the page effectively. I used aria-required="true" on essential form fields to indicate which fields are mandatory, improving form navigation for screen reader users. Additionally, I added aria-hidden="true" to decorative elements, such as icons, so that screen readers can ignore them and focus on the content. I also ensured proper heading structure (h3, h4) to create a logical flow of information, enhancing the experience for users relying on screen readers.

Sitemap Page

Technical Description

The Sitemap Page uses HTML and CSS to create an interactive and visually organized navigation system. The page is structured with an SVG element, where "a" tags wrap around "g" (group) elements, creating clickable areas that represent different pages on the website. Each "g" group contains "rect" elements for the background and foreground of the navigation buttons, ensuring clear separation and interaction. The text inside "text" tags labels each button, improving accessibility. CSS properties like width, height, and transform are used to control the layout and positioning of the buttons. focusable and tabindex attributes ensure keyboard navigation, enhancing accessibility. The responsive layout is managed by using JavaScript to adjust the SVG size dynamically based on the container's width.

Accessibility

To enhance accessibility on the Sitemap Page, I utilized several key techniques. Each a tag includes an aria-label attribute, providing descriptive labels for screen readers, ensuring users with visual impairments understand the navigation options. I also included the role="link" attribute, indicating that these elements are interactive links. The tabindex="0" ensures keyboard users can navigate the page via the Tab key, and focusable="true" is added to SVG elements for better keyboard accessibility. Additionally, each interactive element is wrapped in a g group with a title tag, which provides more descriptive text for assistive technologies.

Content Page

Technical Description

The Content Page was built using HTML for structure and CSS for styling. Key HTML tags such as header, section, and footer were used to organize the page semantically. Headings like h1, h2, and h3 provide a clear content hierarchy, improving readability and accessibility. CSS properties like font-family, color, padding, and margin were applied to ensure proper spacing and legibility. Flexbox was used for layout management, ensuring a responsive design. Images were given a border using border: 2px solid #333 for visual clarity. Overall, the layout is simple, clean, and structured for ease of navigation.

Accessibility

The Content Page has been designed with accessibility in mind. Semantic HTML elements such as header, section, and footer provide a logical structure, improving screen reader compatibility. Headings like h1, h2, and h3 ensure content hierarchy and facilitate navigation for users with assistive technologies. Image borders are used to improve visual clarity, and alt attributes are assumed to be present for all images (though not shown in the code). The page is mobile-friendly, and proper contrast between text and background enhances readability for users with visual impairments. Further accessibility considerations such as keyboard navigation could be explored for enhancement.

Challenges and Lessons Learned

During the development of the User Profile, Sitemap, and Content Pages, several challenges arose. One issue was ensuring consistent design across the pages while adhering to the shared global CSS. This was resolved by closely collaborating with team members to adjust and refine the CSS file, ensuring all components aligned properly. Another challenge was ensuring form validation on the User Profile Page, which was initially cumbersome to implement. Through research, the form was successfully integrated using JavaScript for real-time validation. A key lesson learned was the importance of communication and regular updates to maintain cohesion in design and functionality across the website.

Compliance

The User Profile, Sitemap, and Content Pages were developed with consideration of Janet regulations governing web page publication. These regulations emphasize accessibility, security, and privacy. For accessibility, we ensured proper use of HTML semantic elements, providing clear structure and alternative text for images. The pages also adhere to WCAG 2.1 guidelines by incorporating keyboard navigation and proper contrast for readability. Security measures, such as form input validation, were implemented to protect user data. Privacy concerns were addressed by avoiding unnecessary data collection and clearly stating any data use policies. These pages reflect a commitment to compliance with Janet's web publishing standards.
